: Protector

  VxLang supports packer functionality for user-mode modules (*.EXE/*.DLL/.NET). A packer is a structure that converts an executable to a VxLang file system at pack-time and restores it at run-time. The packer engine provides Up to seven extension modules can be packaged simultaneously, and they can act before the original restore, giving you full control over the packer through events (e.g. adding detections, controlling detections, changing detection messages, etc.). This allows you to easily and freely add more powerful features to develop specialized packers.

Here's how to use it:
vxlang.exe ${file-path}
vxlang.exe ${file-path} --add-on ${add-on-path} --add-key ${key-string}
vxlang.exe ${file-path} --add-data ${raw-data-path} --add-key ${key-string}

--add-on: Compresses the Add-on Module, encrypts it, and includes it in the packing.
--add-data: Compresses the raw data, encrypts it, and includes it in the packing.
--add-key: This command assigns an ID to the add-on module and raw data.
--disable-packer: Disable the packer. In this case, only the obfuscation feature is enabled.

Add-on modules or raw data can be used by calling them through the key value in the add-on.

: ADD-ON(Packer-Plugin)

  The following describes how to write an add-on module.
  Add-On modules in VxLang are developed as DLLs and are notified of packer events through DllMain. Note that add-ons are extensions to the packer shell code, not actual DLL modules. Unlike regular DLLs, there are caveats.

  Each event represents the following meanings.

VXLANG_LOAD_ADDON_EVENT: The event that is notified when the add-on module is loaded for the first time.
VXLANG_START_EVENT: An event that is notified when the packer's source file is ready to run.
VXLANG_TERMINATE_EVENT:The event that is notified when the packer is signaled to terminate (packing failure and add-on termination).

  If you set the return value to FALSE when each event comes in, the packer will terminate and the VXLANG_TERMINATE_EVENT event will be raised.

  These events will be added through updates.

: Axion Add-on Module

  The Axion module is an add-on module to the anti-temper functionality natively supported by VxLang. It includes anti-debugging, detects memory tampering, and performs behaviors such as read protection and pause protection.

  This module is enabled by default for the free version.
