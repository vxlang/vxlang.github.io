: Protector

  VxLang supports packer functionality for user-mode modules (*.EXE/*.DLL/.NET). A packer is a structure that converts an executable to a VxLang file system at pack-time and restores it at run-time. The packer engine provides anti-debugging and patch tampering surveillance by default, and can be customized for your use through extension modules. Up to seven extension modules can be packaged simultaneously, and they can act before the original restore, giving you full control over the packer through events (e.g. adding detections, controlling detections, changing detection messages, etc.). This allows you to easily and freely add more powerful features to develop specialized packers.

Here's how to use it:
vxlang.exe ${file-path} --detect-debug --detect-patch --add-on ${ADD-ON}
--detect-debug: Detect debugging status.
--detect-patch: Detect memory tampering.
--add-on: Add the Add-on module.
--disable-packer: Disable the packer. In this case, only the obfuscation feature is enabled.
